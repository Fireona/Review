Block vs Inline

Most elements we've used so far are block elements, or another way of saying it is their default style is set to "display: block". By default, all block elements will appear on the page stacked on top of each other. Every new block element will create a new line.

On the contrary, inline elements do not create a new line, but appear in line with the elements they are placed next to. A good example would be a link (from using the <a> tags) that is part of a text content. For example:

Blah blah blah blah blah blah blah blah.


As you can see that even though there is a link inside, the line of text is sitting inside as part of the content. Also, the padding and margin properties work differently on inline elements, so make it an effort to not appyl any extra padding or margin on such elements.

The Middle Ground: Inline Block

Inline-block elements behave like inline elements, but have block-style padding and margin. display: in-line block is a useful tool to know about, but generally is overseen by Flexbox when aligning boxes (covered next lesson).

Divs and Spans

div and span elements give no meaning to the content, but are generic boxes that can contain anything. Due to this particular nature, that is what makes both of these elements so widely useful. This allows us to easily plant IDs or classes on them making them easily targettable with CSS. It also allows us to group related elements under a common parent element so we can easily position them on a page.

div is a block element by default. It is mostly used to as a container element to group other elements. It allows us to divide the page into different blocks and apply styling to said blocks.


span is an inline-level element by default. It can be used to group text content and other inline HTML elements for styling. This should only be used when no other semantic HTML element is appropriate.
