The Box Model

The simplest way to structure a webpage is to envision all the elements on the page as a set of building blocks. The simplest shape for us to understand is a rectangular box. You can have one surrounding another, be surrounded by one, next another, or even all three at the same time. Let's use the example below:

CSS 


            * ❴
                outline: 2px solid red;
            ❵

Here we have an image with some content written in HTML, each element surrounded with a 2px solid red line in the shape of a box. When structuring your webpage, each piece of content gets shaped into a rectangular box. When building your webpage, all you are doing is deciding how to lay out all these boxes.

There are many methods to size the boxes to your desire, including the space between each of them. This is done by adding the <padding>, <border>, and <margin> properties to the declaration. To summarize:

Padding
Defines the space between the border of the box and the content.
Border
>
Defines the space between the border and the the margin.
Margin
Defines the space between the margin and adjacent margins of other boxes.

Properties
"box-sizing: content-box"
Works exactly like "border-box", except the limits only go up the the content box. The values on padding border are not included in content-box.
Note: There is no margin-box because margin is not really part of the element itself. Margin is more about the space between adjacent boxes.

  • The box-sizing property in CSS defines how the total size of an element's box is calculated. It can be set to either "content-box" or "border-box".

  • By using "content-box" (the default), the values of "width" and "height" only apply to the content itself – padding and border values are added outside of those dimensions.
  • By using "border-box", the values of "width" and "height" include the content, padding, and border – this means the sum of those values are equal to the "width" and "height" values.

CSS Tricks on Margins

Notes

The margin property defines the outermost property of the box model which creates the space around an element.

Margins are set using lengths, percentages, or the keyword auto, and even negative values. 


                    .box {
                        margin: 0 3em 0 3em;
                    }

margin is a shorthand property and accepts up to four values: 


                    .box {
                        margin: <margin-top> <margin-right> <margin-bottom> <margin-left>
                    }

Each of the margin tags are to have values of some numerical value, percentage, or auto. They are each separated by a single space.

If fewer than four values are defined, the missing values are assumed based on what values are provided. In the example below, the declarations are written differently but both will give the same results: 


                    .box {
                        margin: 0 1.5em;
                    }

                    .box {
                        margin: 0 1.5em 0 1.5em;
                    }
In the first declaration, the values are defined for <top-and-bottom> and <left-and-right>.

auto and Centering

auto tells the browser to define the margin for you. In most cases, the value of auto will be the equivalent to a value of 0 (the initial value for each margin property) or else whatever space is available on that side of the element. However, auto is useful for horizontal centering: 


                    .container {
                        width: 980px;
                        margin: 0 auto;
                    }
In this scenario, two things occur to center this element horizontally within the given space:
  • The element is given a specified width.
  • Left and right margins are set to auto.
Without a defined value for the width, auto will have little to no effect. The reason is width determines how "wide" your element is. Without a value to it, your element will automatically take up the entire space, and leave nothing for your margin (because your element does not include margin). Once a width value is defined, that value is subtracted from the total space available and what remains becomes your margin. This allows the auto value to function and center elements.
  • You can't use margin: auto to center elements vertically by default, because the vertical space in most web layouts is undefined — web pages grow down naturally with content. For vertical centering with auto to work, the element must be inside a parent container with a defined height. Without that constraint, the browser has no vertical space to divide, so auto margins can’t be calculated.

Collapsing Margins

Vertical margins on different elements that touch each other will collapse (overlap). This creates a single margin giving the appearance of whichever margin holds the greater value. This only applies to vertical margins. HTML 


                    <h2>Collapsing Margins</h2>
CSS 

                    h2 {
                        margin: 0 0 20px 0;                    
                    }

                    p {
                        margin: 10px 0 0 0;
                    }
Here we have a scenario where <h2> element has a bottom margin of 20px while our <p> element has a top margin of 10px. When the two margins collapse with each other, only the greater value stands as the margin, that being 20px.

Collapsing may seem unintuitive at first glance, but it is useful. This prevents empty elements from adding extra (usually unwanted) vertical margin space. It also gives way to only have to adjust one margin on a single element. If they did not collapse, we would have to adjust two margins on two elements to maintain consistent spacing. Lastly, collapsing of margins also applies to nested elements as well.
HTML 


                    <h2>Collapsing Marings<h2</h2>
                    <div><p>Example text.</p></div>
CSS 

                    h2 {
                        margin: 0 0 20px 0;
                    }

                    div {
                        margin-top: 40px
                    }

                    p {
                        margin-top: 30px
                    }
Here we have a <p> with a top margin of 30px nested within a <div> element that has a top margin of 40px. We also have an <h2> element with a bottom margin of 20px. If margins did not collapse, we would be looking at a margin of 90px. However, because they do, there is no need to adjust the top margins to remove unneccessary vertical space.

Negative Margins

When interacting with other elements, positive margins push other elements away. Likewise, a negative margin will pull them closer to it.
