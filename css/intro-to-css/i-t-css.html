Syntax

CSS are made of rules composed of selectors, a property-value pair, with a semicolon to separate the declarations of the property-value pair.

With a selector chosen, you begin designating the property to change. In order to do so, we use "❴". This is the beginning of our "declaration", which is made up of a list property-value pair(s).


Properties are the characteristics to style the selectors with. After the property has been chosen, we follow up with a colon to designate value to the property and end with a semicolon. This will end the declaration and apply it to the chosen selectors. We begin the next line if we have any more styles to apply, if not then end the declaration with "❵".


Selectors

Selectors are as implied. They only "select" HTML elements to apply CSS rules to.

Universal Selector

The universal selector is designated with an asterisk. Usage of the universal selector applies to elements of every type, meaning the entire document. 

                CSS
                
                    * {
                        color: purple;
                    }

Type Selectors

Type selectors will select all elements of a given element type. The syntax is simply the name of the element. 

                CSS
                
                    p {
                        color: purple;
                    }

Class Selectors

Class selectors are a way of selecting particular type selectors of a given attribute name. This is done by adding in a class attribute to the HTML element. 

                HTML
                
                    <p class="blue"> This is a blue paragraph.
                        <p class="blue"> This is also a blue paragraph.
                            <div class="red"> This is a red paragraph.
                
            
                CSS
                
                    .blue {
                        color: blue;
                    }

In this scenario, class selectors are designated with a period preceding the class selector.

ID Selectors

ID Selectors are similar to class selectors. They select elements with the given ID, another attribute placed in an HTML element. The only caveat is each element can only have one ID. It cannot be repeated on a single page and should not have any whitespace. 

                HTML
                
                    <p class="blue"> This is a blue paragraph.
                        <p class="blue"> This is also a blue paragraph.
                            <p class="red"> This is a blue paragraph.
                
            
                CSS
                
                    #red {
                        color: red;
                    }

In this example, the ID selector is designated with a hashtag preceding the ID selector name.

Grouping Selector

There are times when multliple groups of elements that share the same declarations. To combine selectors, use "," to separate the elements. 

                CSS
                
                    .title {
                        color: black;
                        background-color: white;
                        /* several unique declarations */
                    }
            
                    .headline {
                        color: black;
                        background-color; white;
                        /* several unique declarations */
                    }

In this example, we see that both title and headline both have different class selectors but have the some same declarations. If we want to combine these two selectors, it would be written as below: 

                CSS
                
                    .title, .headline {
                        color: black;
                        background-color; white;
                    }
            
                    .title {
                        /* several unique declarations */
                    }
            
                    .headline {
                        /* several unique declarations */
                    }

While this method and former method both achieve the same results, it is more efficient written this way and easier to edit both simultaneously.

Chaining Selectors

Another method of selecting multiple selectors is with the chain selector. To use a chain selector, write the two selectors as usual, but instead of a ,, use a period without a space to separate them. 

                HTML
                
                    <div class="electronic edm">Electronic Music</div>
                        <p class="electronic house">This is about House Music!</p>
                
            
                CSS
                
                    .electronic.edm {
                        color: red;
                    }

As we can see from the example above, the declaration applies to both elements that the "electronic" and "edm" classes. However, chain selectors will only work with any combination of selectors except for more than one type selector. You can also chain a class selector with an ID selector. 

                HTML
                
                    <div class="electronic edm">Electronic Music</div>
                        <p class="electronic house" id="chill">This is about House Music!</p>
                
            
                CSS
                
                    .electronic.edm {
                        color: red;
                    }
        
                    .electronic#chill {
                        color: blue;
                    }

In general, you can not chain more than one type selector because an element can't be two elements at once. Chaining two type selectors like <div> and <p> would give us the selector <divp> instead. This does not work because there is no <divp> element as it does not exist in HTML. Writing it as <div.p> does not work either because that tells CSS to look for the element <div> with the "p" class. If you need to apply particular declarations on two type selectors, use the grouping selector instead.

Descendant Combinator

The Descendant Combinator is one of four combinators. It is represented with a single space between selectors. This selector only selects the last element written if they also have an ancestor that matches the preceding selector. 

                HTML
                
                    <div class="parent">This is a parent.<div>
                        <div class="child">This is a child.<div>
                            <div class="child">This is a child.<div>

                    <div class="child">This is a child.<div>
                
            
                CSS
                
                    .parent .child {
                        /* Some declarations */
                    }

In the example above, we see that one <div> has the class "parent" and three of them have the class "child". However, the declarations would only apply to both nested elements within the class "parent". This is because regardless of how deeply nested the element is, if the last element written element is only selected is any prior elements match with the preceding selectors.

Properties

Typogpraphy Properties
color
Changes text colour. Values are names of colours such as "red", keywords like "transparent", and even RGB, HEX, HSL values.
background-color
Changes background colour. Values are names of colours such as "red", keywords like "transparent", and even RGB, HEX, HSL values.
font-family
Changes the font used. Can be a single value, or separate multiple values with comma. If the browser does not support the font, it will cycle on to the next available font. A fallback in inserted in the end, after your desired font, to be used as a fallback.
font-size
Changes size of the font. Values are a number followed by "px", such as "22px".
font-weight
Changes the boldness of the font. So long as the font supports the boldness, the weight of the font will be changed. Values range from 1px to 1000px. 700px is the equivalent of "bold".
text-align
Aligns text. Values are similar to word processors, such as "text-align: center".

Adding CSS to HTML

External CSS

To implement CSS to HTML externally, we have to create a CSS file within the same directory. This is then linked in the HTML opening and closing <head> tags with a void <link> element. 

                HTML
                
                    <head>
                        <link rel="stylesheet" href="styles.css">
                    </head>
                
            
                
                    div {
                        color: white;
                        background-color: black;
                    }
            
                    p {
                        color: red;
                    }

After adding a void <link> element within the <head> tags of the HTML file, we still have to add an href attribute. This leads the link to the CSS file, which could be an absolute URL, or in this scenario a relative location of the HTML file. The "rel" attribute is required as it specifies the relationship between the HTML file and the linked file.

Using the example above, the CSS file styles.css is already filled with a type selector, <div> with a declaration block of a white text and black background surrounded by opening and closing curly brackets. The name of the file doesn't matter, but generally are named "style" or "styles".

External CSS method is used to keep the files smaller and makes it look cleaner. Also, because you only need to edit CSS in one place, it is easier to keep track of when dealing with multiple pages of similar styles.

Internal CSS

Internal CSS involves adding CSS within the HTML file itself, only having a need for one file. With this method, all the rules are placed within a pair of opening and closing <style> tags. This "style" block is then placed inside the opening/closing <head> tags of our HTML file. Since everything is in the HTML file, a <link> element is no longer needed that the external method uses. Afterwards, the same declaration blocks in CSS are written the same way in the HTML file. 

                HTML
                
                    <head>
                        <style>
                            div {
                                color: white;
                                background-color: black;
                            }
                        
                            p {
                                color: red;
                            }
                        </style>
                    </head>
                    <body>
                    ...
                    </body>

Inline CSS

Inline CSS allows you to add styles directly to HTML elements, but this method is generally avoided. The way this works is by using the "style=" attribute inside the element tags. Within the quotations, you write the declarations the same way. 

                HTML
                
                    <body>
                        *ltdiv style="color: white; background-color: black;">...</div>
                    </body>

This method is generally only used to style a single element. However, due to the nature of this method, it generally is avoided to keep things clean. Once you begin to add more declarations, the HTML starts to get messy and difficult to navigate. If you want other elements to have the same declarations, then you'll have to write or copy those as well thus making it feel bloated. This method also overrides the other two methods so use it sparringly if never at all.
