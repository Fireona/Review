Growing and Shrinking

Overview

    The three properties that are defined by the flex shorthand.

Flex Shorthand

When using the flex declaration, it is actually a shorthand for three properties that affect how flex items size themselves in the flex container. The flex is a shorthand for:

  • flex-grow
  • flex-shrink
  • flex-basis



In the diagram above, we see <div> is defined by flex: 1. This is in reality a shorthand for:

    flex: 1 1 0
  • flex-grow: 1
  • flex-shrink: 1
  • flex-basis: 0
In many cases, when the flex shorthand is defined by one value, it only affects the property flex-grow. If we use the example from our previous lesson, we changed our element to become a flex container. Following up from that, we also applied flex: 1, which is actually a shorthand to write flex: 1 1 0.

The default values for flex is:

    flex: 0 1 auto
  • flex-grow: 0
  • flex-shrink: 1
  • flex-basis: auto
What this means is:
  • Items do not grow.
  • The items can shrink.
  • Their sizes are based on content, width, or min-content.


Flex-grow

flex-grow expects a single number as its value, which determines the flex item's "growth factor". In our earlier example, when we declared flex: 1 to all <div>s in the container, we told all divs to grow the same rate. If we wrote for one div to have flex-grow: 2, then that div would grow twice as large.

In this link, you can see there is flex shorthand applied, with default values for flex-shrink and flex-basis. By changing the value of flex-grow, you can see that the div expands horizontally accordingly to the value specified with respect to the other <div> elements.

Summary

  • Affects how flex-items grow.
  • Value of 0 prevents growth, any greater value will allow the item to grow.
  • flex: 1 is shorthand for flex: 1 1 0.


Flex-shrink

Similar to flex-grow, flex-shrink sets how a flex item "shrinks". This only functions when the size of all three flex items becomes larger than the parent. Lets make a scenario using the same example above. If we had the three <div>s to have a specified equal width and that value exceeded our container's. What would happen is the three flex items would have to shrink in order to fit in the flex container.

The default shrink factor is flex-shrink: 1. If you write flex-shrink: 0, then the items will not shrink at all. By applying a higher value to flex-shrink, the items will shrink at a greater rate.

In this scenario, we have three <div>s that have a specified width. If we shrink the window of the browser, .two will never become smaller than the declared width: 250px;. This applies even when flex-grow stated for all <div>s to be grow at an equal rate.

An important fact to notice here when specifying flex-grow or flex-shrink, flex items do not necessarily respect any given values to width. In the above example, three <div>s are given width: 250px;, but they are "flexed" to fill the container when big enough. The opposite will occur when scenario is reversed. If the container is too small,the default behavior is for the flex items to shrink.

    Summary:
  • Only functions when the total size of all flex items exceed the container's.
  • Similar to flex-grow, but sets the rate at which a flex item shrinks.
  • Default value for flex-shrink is 1.
  • Giving a value of "0" to flex-shrink means the item can not shrink at all.
  • Items will not shrink below a declared width property, but will still stretch to fill the available space.


Flex-basis

flex-basis sets the initial size of a flex item. This sets as the baseline for any flex-grow or flex-shrink values. The shorthand value defaults to flex-basis: 0%. When using the value auto for flex-basis tells the item to check for a width declaration.

    In the example we set in Flex-Shrink, the value for flex-basis was set to auto because if left at a value of 0, then the <div>s would ignore the container's width and then shrink evenly.

Note: There is a difference between the default value of flex-basis and the way flex defines it if flex-basis was not declared. The actual default value for flex-basis is auto, but when we specify flex: 1, it automatically interprets it as flex: 1 1 0. If we only want to adjust an item's flex-grow, we must do so without the shorthand.

    Summary:
  • flex-basis sets the initial size of an item.
  • flex is shorthand for flex: 0 1 auto. This does not let items grow.
  • flex: 1 is shorthand for flex: 1 1 0. This lets items grow.
  • If we only want to adjust an item's flex-grow, we can do it directly without the shorthand.
  • An alternative is to write the entire value shorthand flex: 1 1 auto, the equivalent of flex: auto.
  • Valid values are:
    • auto
    • em
    • %
    • min-content
    • min-width (change to min-height when main axis is vertical)
    • max-width (change to max-height when main axis is vertical)


    flex-auto

    When auto is defined as as flex keyword, it is the same property values of flex-grow: 1, flex-shrink: 1, and flex-basis: auto. This is also the shorthand for flex: 1 1 auto.

    A note to be aware of is flex: auto is not the default value when using the flex shorthand.

    Summary:

    • When applied as the keyword (value) to flex, it is a shorthand for writing flex: 1 1 auto (or in more expanded form flex-grow: 1, flex-shrink: 1, and flex-basis: auto).


    W3C's Flex Section

    Below are four flex values that give the most commonly-desired effects:

    flex: initial
    Equivalent to flex: 0 1 auto. Sizes the item based on their width/height properties. (If the item's main size property computes to auto, this will size the flex item based on its contents). Makes an item inflexible when there is positive free space, but allows it to shrink to its minimum size where there is insufficient space. Alignment abilities or 'auto' margins can be used to align flex items along the main axis.
      Summary:
    • flex-grow: 0 prevents the item from growing.
    • flex-shrink: 1 allows the item to shrink if necessary.
    • flex-basis: auto means the item's starting size is based on the width, the content, or default behavior.
    • Behaves like a block-level element. Takes up only as much space as needed, shrinks if needed, but doesn't grow to fill extra space.

    flex: auto
    Equivalent to flex: 1 1 auto. Sizes the item based on their width/height properties, but makes them fully flexible allowing them to absorb any free space along the main axis. If all items are either flex: auto, flex: initial, or flex: none, then any positive free space after the items have been sized will be distributed evenly to the items with flex: auto.
      Summary:
    • flex-grow: 1 allows the item to grow.
    • flex-shrink: 1 allows the item to shrink if necessary.
    • flex-basis: auto means the item's starting size is based on the width, the content, or default behavior. Due to variances in browsers, the value of "auto" does not equate to "0" necessarily.
    • Very flexible. Item grows and shrinks based on available space, and respects content size.
    • Commonly used when you want items to scale proportionally but also let content influence size.

    flex: none
    Equivalent to flex: 0 0 auto. Sizes the item based on their width/height properties, but makes them fully inflexible. Similar to flex: inital, but flex items are not allowed to shrink even when overflowing.
      Summary
    • flex-grow: 0 prevents the item from growing.
    • flex-shrink: 0 prevents the item from shrinking.
    • flex-basis: auto means the item's starting size is based on the width, the content, or default behavior.
    • Completely rigid. Keeps items their natural size no matter what.
    • Useful when you want a fixed-size item inside a flexible layout (like a sidebar or logo).

    flex: <postive-number>
    Equivalent to flex: "#>0" 1 0. Makes the flex item flexible and sets the flex basis to zero, resulting in an item that receives the specified proportion of the free space in the flex container(?). If all items in the flex container use this pattern, their sizes will be proporitonal to the specified flex factor.
      Summary
    • flex-grow: #>1 allows the item to grow at specified value.
    • flex-shrink: 1 allows the item to shrink if necessary.
    • flex-basis: 0 means the item size starts at 0 width, grows from there.
    • Used to evenly or proportionally divide space among items.


    MDN's Documentation on Flex

    Default value for using the shorthand flex is flex-grow: 0, flex-shrink: 1, flex-basis auto.

    Default values for flex-grow:

    • Any number equal or greater than 0.
    • inherit
    • initial
    • revert
    • revert-layer
    • unset

    Default values for flex-shrink

    • Any number equal or greater than 0.
    • inherit
    • initial
    • revert
    • revert-layer
    • unset

    Default values for flex: basis

    • em
    • px
    • %
    • auto
    • Intrinsic Sizing Keywords:
      • max-content
      • min-content
      • fit-content
    • Automatically Sized Based on Flex Item's Content:
      • content
    • Global Values:
      • inherit
      • initial
      • revert
      • revert-layer
      • unset

    When using valid values paired the flex shorthand, with:

    1. One Value
      • Changes the flex shorthand to flex-grow: 'value' or flex: 'value' 1 0. It will never affect flex-shrink.
      • If defined by a valid value for flex-basis, changes the flex shorthand to flex: 0 1 'value'.
    2. Two Values
      • First value changes the flex shorthand to flex-grow: 'value' or flex: 'value' 1 0. It will never affect flex-shrink.
      • Second value must be a valid value for flex-basis. This changes the flex shorthand to flex: 0 1 'value'.
    3. Three Values
      • If all values are valid values, then changes the flex shorthand to flex: 'first value' 'second value' 'third value', affecting the properties of flex-grow, flex-shrink, and flex-basis respectively.
