Axes


Flexbox can work horizontally or vertically, but some rules change depending on the orientation we are working in. The default direction for flex container is horizontal, row, but it can be changed to vertical, column. The declaration wil look like this: 

                CSS
                
                    .flex-container {
                        flex-direction: column;
                    }

Think of your containers as having two axes: the main axis and the cross axis. Direction of the axes change as a resulting effect when flex-direction changes. In most use cases, flex-direction: row puts the main axis horizontal (left to right), and column puts the main axis vertical (top to bottom).

In our first example from Intro to Flexbox, our parent <div> was given the display: flex property and the items were then arranged horizontally. Default setting for flex direction is flex-direction: row. In this example, if we apply the property flex-direction: column instead, the items will stack vertically.

From this example, flex-direction: column will not work if the shorthand flex: 1 was used. Remember, flex: 1 is shorthand for flex: 1 1 0. From this, the flex-basis states the initial size of an item, which is defined as "0" in this scenario meaning to "grow" from nothing. Empty <div>s, by default, have 0 height. With a lack of content or "room to grow", the items remain empty, giving the appearance of the borders only. Even with a declared property of height: 80px, our flex-basis property still tells us to start from 0. Our height: 80px instead then behaves as a height limiter.

    Summary for flex: 1 and why it doesn't work.
  • flex: 1 stands for flex-grow: 1, flex-shrink: 1, flex-basis: 0. flex-basis: 0 says the initial size of the item starts at 0.
  • Height declarations will be ignored in favor of flex-basis when using flex-basis: 0.Any declared heights will not be used as a starting point, but act as a height limiter.
  • Empty <div>s have a height of 0 by default. With no content to fill the item, only borders remain visible.
    Summary for flex: 1 1 auto.
  • flex: 1 1 0 stands for flex-grow: 1, flex-shrink: 1, flex-basis: auto. flex-basis: auto means "Start from where you have been told or from nothing".
  • With height: 80px declared, flex-basis' initial size starts at 80px.
  • With our <div>s being told by flex-basis: auto to start growing at our declared height: 80px, our items appear "filled" or "stretched" as boxes stacked on top another.

Alternatively, another option is to achieve a similar result is to apply a height on the parent container or to use flex-grow: 1 instead.

Note: When changing the flex-direction to column, the flex-basis refers to height instead of width (use the visual in the example above).

Default behaviour is flex-direction: row which arranges things horizontally. This often functions nicely without changing other details in CSS is because block-level elements default to the full width of the parent. Changing things to vertical with flex-direction: column adds complexity. Block-level elements default to the height of their content, and in this case, there is no content.


Questions

How do you make flex items arrange themselves vertically instead of horizontally?

By applying the declaration flex-direction: column to the flex container.

In a column flex-container, what does flex-basis refer to?

height.

In a row flex-container, what does the flex-basis refer to?

width.

Why do the previous two questions have different answers?

They have different answers because of the direction of the content is arranged. Flexbox lays out items along a main axis, which is defined by the flex-direction.
  • In a row direction, the main axis is horizontal, so flex-basis controls width.
  • In a column direction, the main axis is vertical, so flex-basis controls height.
flex-basis always defines the initial size of a flex item along the main axis, which is why it refers to different physical properties depending on the layout direction.
