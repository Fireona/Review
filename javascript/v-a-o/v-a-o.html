How to run JavaScript code

All JavaScript written in the majority of the course will be run via browser. Later lessons in the Foundations and NodeJS path will show how to run JavaScript outside of the browser environment. This means that we will let the browser execute the code, even if it comes from a file we created.

Create a new HTML file and add the following script: 

                        <script>
                        
                            console.log("Hello, World!")
                        
                        </script>
When you inspect the page and enter the Console tab of the DevTools, you will see the script pop up.

console.log is a command that directly prints something into the developer console of the browser.

Another method to include JavaScript into a webpage is with an external script. This is similar to linking an external CSS document to your website. 

                    
                        <script src="javascript.js"></script>

Just like how CSS documents have the extensions .css for stylesheets, JavaScript has the extension .js. External JavaScript files are used for more complex scripts.

External JavaScript files can be named freely into almost anything. So long as they have the .js extension, it will function. JavaScript files can be named anything, and can even be nameless (just .js).

Variables

Variables are the building blocks of any program. Think of them as "storage containers" for data in your code.

Variables can be declared using the let keyword. 

                    <script>
                    
                        let name = "John";
                        let surname = "Doe";

                        console.log(name);
                        console.log(surname);
                    
                    </script>

Inputting the above script will enter two logs into the console, one for the name and another for surname.

Variables can also be re-assigned. 

                    <script>
                    
                        let age = 11;
                        console.log(age);

                        age = 54;
                        
                        console.log(age);
                    
                    </script>

In the above script, the second declaration of age has omitted the variable let. This is because the variable had already been declared earlier, and you do not need it again when re-assigning values to variables.

When we have a constant that is never changed (like pi), we can use the keywordconst instead. 

                    <script>
                    
                        const pi = 3.14;
                        pi = 10;

                        console.log(pi);
                    
                    </script>

When inputting the above script, an error occurs. It does not reach the console log. This method of creating errors helps identify the location of said error and what is wrong.

In short, there are two methods of declaring a variable:

  • let, which can be re-assigned.
  • const, which can not be re-assigned.

In the past, the original method of declaring variables was the keyword var. However, there were issues when used that were fixed when let and const were introduced. Today, var is no longer used but you may still encounter it in other areas.

Variable Naming

There are two limitations on variable names in JavaScript:

  • The name must contain only letters, digits, or the symbols $ or _.
  • The first character must not be a digit. 

                    <script>
                    
                        let userName;
                        let test123;
                    
                    </script>

Words go one after another, with each word starting with a capital letter with the exception of the first word. 

                    <script>
                    
                        thisVeryLongSetOfWordsStrungTogether
                    
                    </script>

The dollar sign $ and the underscore _ can also be used in names. They are regular symbols, just like letters, without any special meaning. 

                    <script>
                    
                        let $ = 1;
                        let _ = 2;

                        console.log($+_);
                    
                    </script>

Two variables that are spelt the same but have different letter case used are considered to be different variables. 

                    <script>
                    
                        fire;
                        FIRE;
                    
                    </script>

Non-Latin characters such as Cyrillic letters, Chinese logograms, and etc. are technically allowed, but there is an international convention to use English in variable names. This also helps to preserve the work through conventional means and make it easier to use. 

                    <script>
                    
                        let имя = '...';
                        let 我 = '...';
                    
                    </script>

There are also sets of reserved words that can not be used. These are keywords that are used by the language itself. Some examples being let, class, return, and function. Attempting to use these words as variables will result in errors. For example: 

                    
                    <script>
                    
                        let let = 5;
                        let return = 10;
                    
                    </script>

Constants

To declare a constant, use const. 

                    <script>
                    
                        const pi = 3.14
                        const g = 22/7
                    
                    </script>

Variables declared using const are called constants and can not have values re-assigned. Any attempt doing so will result in an error.

A practice between developers is to assign constants to values that are difficult to remember. These constants are named using capital letters and underscores. Some examples are colors in hexadecimal values. 

                    <script>
                    
                        const COLOR_RED = "#F00";
                        const COLOR_GREEN = "#0F0";
                        const COLOR_BLUE = "00F";
                        const COLOR_ORANGE = "#FF7F00";

                        // ...when we need to pick a color
                        let color = COLOR_ORANGE;
                        alert(color);
                    
                    </script>

In this scenario, using the const COLOR_ORANGE is much easier to handle than it's hexadecimal value "#FF7F00". It is easier to remember, type, and recognize.

There are rules to follow when using capitals in a constant. Some values are assigned before execution (before a page is loaded). There are also values are calculated first during the execution, but do not change after the initial assignment. 

                    <script>
                    
                        const pageLoadTime = 
                    
                    </script>

The value of pageLoadTime is not known before a page is loaded, so it is named normally. However, because it is still a constant as it's value does not change once assigned. This means that capital-named constants are used for aliases that are "hard-coded" values.

Name things right

Any variable should have an alias that should be easily recognizable that describes the data it represents. A good example would be how easy a variable is recognized after doing something else for a period of time.

A general guideline:

  • Use human-readable names like userName or shoppingCart.
  • Avoid using abbreviations or short names such as x, b, c, etc. unless you know what you are doing.
  • Make names maximally descriptive and concise. Some bad examples are data and value because they don't tell use anything. It is only acceptable if the context makes it obvious what data or value they are referring to.
  • Terms should be agreed upon in mind and with the team. If a site visitor is called user then we should name related variables as currentUser or newUser rather than something oblivious as currentVisitor or newManInTown.

W3Schools' on JavaScript Numbers

There are two types of numbers in JavaScript, one with or without decimals. 

                    <script>
                    
                        let x = 3.14;
                        ley y = 3;
                    
                    </script>

Extra large or extra small numbers can be written with scientific notation. 

                    <script>
                    
                        let x = 123e5; // 12300000
                        let y = 123e-5; // 0.00123
                    
                    </script>

JavaScript Numbers are Always 64-bit Floating Point

JavaScript stores numbers as double precision floating point numbers, following the internatioal IEEE 754 standard.

Floating point numbers are like scientific notation. 

                    
                        1.23 x 10^5 = 123,000

1.23 is the "fraction" (also called mantissa or significand) and the 5 is the exponent (dictates decimal position).

The way the number is stored in JavaScript is by using 64 bits (binary), or double precision (single precision is 32 bits).

  • Bit 0 stores the integer sign telling if it is positive or negative.
  • Bits 1-11 stores the exponent, telling how large or small a number is.
  • Bits 12-63 stores the fraction or significand.

Writing in 64 bit double precision allows for 15-17 digits.

The IEEE 754 is an international standard that defines how floating point numbers are stored, what the bits mean, and how to handle special values like Infinity and NaN. This standard is followed by JavaScript and other languages such as C, Python, and Java.

Integer Precision

Integers, numbers without a period or exponent notation, are accurate up to 15 digits. 

                    <script>
                    
                        let x = 999999999999999; // x will be 999999999999999
                        let y = 9999999999999999; // y will be 10000000000000000
                    
                    </script>

Floating Precision

Due to the nature of the binary system, floating point arithmetic is not always 100% accurate. 

                    <script>
                    
                        let x = 0.2 + 0.1;
                    
                    </script>

Writing the script above will not have the evaluation of "0.3" but rather a very close value to it.

To prevent this problem, we have add some multiplication and division processes to work properly. 

                    <script>
                    
                        let x = (0.2 * 10 + 0.1 * 10) / 10;
                    
                    </script>

Due to the nature of JavaScript (binary system), the process is a bit more complicated. Each variable is changed to their binary form, and 0.1 and 0.2 do not have binary values that perfectly represent them. The end result of the evaluation process gives us "0.30000000000000004" but when returning as a decimal string (a set of value to be displayed as is), we get "0.3". This is because JavaScript follows a rule:

"When converting a number to a string, use the shortest decimal representation that will round-trip correctly back to the same binary value."

This means that although the binary representation is "0.30000000000000004", the shorest decimal string that best represents is "0.3" (this also means when converting back to binary, this value is can also return the binary value "0.30000000000000004").

JavaScript will also provide shorter decimal strings in the console, unless explicity told so. This is to keep all data as human-readable possible. Therefore, all numerical values will be given a considerably shorter decimal string, but one that is still accurate to the initial input.

Adding Numbers and Strings

When adding two numbers, the result will be a number. Example. 

                    <script>
                    
                        let x = 10;
                        let y = 20;
                        let z = x + y;
                    
                    </script>

However, take caution as the + operator as both for additon and concatenation. When adding two strings, the result will be a string concatenation. Example. 

                    <script>
                    
                        let x = "10";
                        let y = "20";
                        let z = x + y;
                    
                    </script>

If you add a number and a string, the result will be a string concatenation. Example. 

                    <script>
                    
                        let x = 10;
                        let y = "20";
                        let z = x + y;
                    
                    </script>

Likewise, if you add a string and a number, the result will be a string concatenation. Example. 

                    <script>
                    
                        let x = "10";
                        let y = 20;
                        let z = x + y;
                    
                    </script>

A common mistake to make is thinking the example below to result in 30. Example. 

                    <script>
                    
                        let x = 10;
                        let y = 20;
                        let z = "The result is: " + x + y;
                    
                    </script>

The truth is, the string only displays the value inside as is. Just like the previous examples with adding strings and numbers or vice-versa, the result is a string concatenation.

Another common mistake is in this example, the answer we expect is 102030, but is not quite so. 

                    <script>
                    
                        let x = 10;
                        let y = 20;
                        let z = "30";
                        let result = x + y + z;
                    
                    </script>

Like in basic mathematics, JavaScript will follow the order of operations. So x and y will merge first becoming "30". However, because z is a string cantenation, the result be as such becoming "3030".

    TL:DR;
  • JavaScript interpreter works from left to right.
  • First 10 + 20 is added because x and y are both numbers.
  • Then 30 + "30" is concatenated because z is a string.

Numeric Strings

JavaScript strings can have numeric content. 

                    <script>
                    
                        let x = 100; // x is a number

                        let y = "100"; // y is a string
                    
                    </script>

JavaScript will try to convert strings to numbers in all numeric operations. For instance, this example for divison. 

                    <script>
                    
                        let x = "100";
                        let y = "10";
                        let z = x / y;
                    
                    </script>

For multipying, this example works. 

                    <script>
                    
                        let x = "100";
                        let y = "10";
                        let z = x * y;
                    
                    </script>

For subtraction, this example works. 

                    <script>
                    
                        let x = "100";
                        let y = "10";
                        let z = x - y;
                    
                    </script>

But this does not work. 

                    <script>
                    
                        let x = "100";
                        let y = "10";
                        let z = x + y;
                    
                    </script>

Do not forget that when using the + operator on strings, the result ends in a string concatenator.

NaN - Not a Number

NaN is a JavaScript reserved word indicating that a number is not a legal number. Attempting to do arithmetic with a non-numeric string will result in NaN. For example. 

                    <script>
                    
                        let x = 100 / "Apple";
                    
                    </script>

However, if the string is numeric string, the result will be a number. Like so. 

                    <script>
                    
                        let x = 100 / "10";
                    
                    </script>

The global JavaScript function isNan() can be used to find out if a value is not a number. In this case: 

                    <script>
                    
                        let x = 100 / "Apple";
                        isNaN(x);
                    
                    </script>

Be careful when using NaN. If used as a variable, the result will also ne NaN. Such as this. 

                    <script>
                    
                        let x = NaN;
                        let y = 5;
                        let z = x + y;
                    
                    </script>

Or it could end up as a concatenation. Like in this scenario. 

                    <script>
                    
                        let x = NaN;
                        let y = "5";
                        let z = x + y;
                    
                    </script>

NaN is a number: typeof NaN returns number. Like in here 

                    <script>
                    
                        type of NaN;
                    
                    </script>

Oddly enough, while NaN stands for "Not a Number", JavaScript still groups it under the "Numbers" family. The reason why is because under the IEEE 754 standard for representing numbers, NaN is considered to be a unique number type. In summary, the reasons why NaN:

    Is Not a Number:
  • It has no meaningful or distinct numerical value.
  • Arises from operations like 0/0, Math.sqrt(-1), or "hello" * 5.
    Is a Number:
  • Under the IEEE 754 standards for numbers, NaN is considered a valid value of the Number type.
  • It is represented as a reserved bit pattern within floating-point numbers.
  • typeof NaN === "number" confirms this classification in JavaScript.

Infinity

Infinity (or -Infinity) is the value JavaScript will return if you calculate a number outside the largest possible number. Like so. 

                    <script>
                    
                        let myNumber = 2;
                        // Execute until Infinity
                        while (myNumber != Infinity) {
                            myNumber = myNumber * myNumber;
                        }
                    
                    </script>

Division by zero also generates Infinity. Like this. 

                    <script>
                    
                        let x = 2 / 0;
                        let y = -2 / 0;
                    
                    </script>

Infinity is a number: entering typeof Infinity will return number. For example.

Hexadecimal

JavaScript interprets numeric constants as hexadecimal if they are preceded by 0x. Like this. 

                    <script>
                    
                        let x = 0xFF;
                    
                    </script>
    Never write a number with a zero at the front, i.e 07.
  • Some JavaScript versions interpret numbers as octal if written with a leading zero.

By default, JavaScript displays numbers as base 10 decimals. However, by using the tag toString(), you can display numbers from base 2 to base 36.

  • Binary is base 2.
  • Octal is base 8.
  • Decimal is base 10.
  • Hexadecimal is base 16.
This example demonstrates the use case. 

                    <script>

                        myNumber.toString(16);
                        myNumber.toString(12);
                        myNumber.toString(10);
                        myNumber.toString(8);

                    </script>

JavaScript Numbers as Objects

In most cases, JavaScript numbers are primitive values created from literals. 

                    <script>
                    
                        let x = 123;
                    
                    </script>
  • Literals are the raw input, i.e 42, "hello".
  • Primitive values are core data types the literals create (number, string, etc.)
  • Objects are containers that group values and behaviors.

Numbers can also be defined as objects with the keyword new. Consider this. 

                    <script>
                    
                        let x = 123;
                        let y = new Number(123);
                    
                    </script>

Here we see x is the number while y is the object. The primitive value (number) here is x and 123 is the number literal. Meanwhile, in the second line of variable declaration, y is the number object.

While we can create Number objects, it is best advised not to do so. The keyword new complicates codes and slows down execution speed, causing Number Objects to produce unexpected results.

In the following example, when using the == operator, x and y are equal. 

                    <script>
                    
                        // x is a number;
                        let x = 500;

                        // y is an object
                        let y = new Number (500);
                        document.getElementById("demo").innerHTML = (x==y);
                    
                    </script>

However, in the example below, x and y are not equal when using the === operator. 

                    <script>
                    
                        // x is a number
                        let x = 500;

                        // y is an object
                        let y = new Number(500);
                        document.getEleemntById("demo").innerHTML = (x===y);
                    
                    </script>

Back then in JavaScript, the == operator was used to compared two values for equality. However, doing so will allow JavaScript to convert values to be of the same type before comparing them. This lead to unexpected conversions behind the scenes.

This brought forth the operator ===. This operator is used now to compare to values for equality, but it does not convert any values into any type at all. It only takes the values as is.

In the example below, is (x == y) true or false? 

                    <script>
                    
                        // x is an object
                        let x = new Number(500);

                        // y is an object
                        let y = new Number(500);

                        document.getElementById("demo").innerHTML = (x==y);
                    
                    </script>

In the example below, is (x === y) true or false? 

                    <script>
                    
                        // x is an object
                        let x = new Number(500);

                        // y is an object
                        let y = new Number(500);

                        document.getElementById("demo").innerHTML = (x===y);
                    
                    </script>

JavaScript will always return false value when comparing two objects.

Types of Numbers

There are two numbers of numbers.

  • Integers are numbers without a fractional part. They can be positive or negative. E.g., 10, 400, -5.
  • Floating point numbers (floats) have decimal points and decimal places. E.g., 12.5, 3.14, 56.77.

There are also different types of number systems. Decimal is base 10 (uses 0-9 in each digit), but there's also:

  • Binary - The lowest level language of computers; 0s and 1s.
  • Octal - Base 8, uses 0-7 in each digit.
  • Hexadecimal - Base 16, uses 0-9 and then a-f in each digit. Commonly found in colour settings in CSS.

JavaScript has only one data type for numbers, integers and decimals, and that is Numbers. This means when dealing with Numbers type data, you must handle them in the same way, with Numbers type data as well.

                    </script>

Enter the values from the example above into the console (F12) and it should return the same values when entering the constants.
