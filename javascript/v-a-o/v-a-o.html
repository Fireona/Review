How to run JavaScript code

All JavaScript written in the majority of the course will be run via browser. Later lessons in the Foundations and NodeJS path will show how to run JavaScript outside of the browser environment. This means that we will let the browser execute the code, even if it comes from a file we created.

Create a new HTML file and add the following script: 

                        
                            <script>
                                console.log("Hello, World!")
                            </script>
When you inspect the page and enter the Console tab of the DevTools, you will see the script pop up.

console.log is a command that directly prints something into the developer console of the browser.

Another method to include JavaScript into a webpage is with an external script. This is similar to linking an external CSS document to your website. 

                    
                        <script src="javascript.js"></script>

Just like how CSS documents have the extensions .css for stylesheets, JavaScript has the extension .js. External JavaScript files are used for more complex scripts.

External JavaScript files can be named freely into almost anything. So long as they have the .js extension, it will function. JavaScript files can be named anything, and can even be nameless (just .js).

Variables

Variables are the building blocks of any program. Think of them as "storage containers" for data in your code.

Variables can be declared using the let keyword. 

                    <script>
                    
                        let name = "John";
                        let surname = "Doe";

                        console.log(name);
                        console.log(surname);
                    
                    </script>

Inputting the above script will enter two logs into the console, one for the name and another for surname.

Variables can also be re-assigned. 

                    <script>
                    
                        let age = 11;
                        console.log(age);

                        age = 54;
                        
                        console.log(age);
                    
                    </script>

In the above script, the second declaration of age has omitted the variable let. This is because the variable had already been declared earlier, and you do not need it again when re-assigning values to variables.

When we have a constant that is never changed (like pi), we can use the keywordconst instead. 

                    <script>
                    
                        const pi = 3.14;
                        pi = 10;

                        console.log(pi);
                    
                    </script>

When inputting the above script, an error occurs. It does not reach the console log. This method of creating errors helps identify the location of said error and what is wrong.

In short, there are two methods of declaring a variable:

  • let, which can be re-assigned.
  • const, which can not be re-assigned.

In the past, the original method of declaring variables was the keyword var. However, there were issues when used that were fixed when let and const were introduced. Today, var is no longer used but you may still encounter it in other areas.

JavaScript.info on Variable Naming

There are two limitations on variable names in JavaScript:

  • The name must contain only letters, digits, or the symbols $ or _.
  • The first character must not be a digit. 

                    For Example:
                    
                        let userName;
                        let test123;

Words go one after another, with each word starting with a capital letter with the exception of the first word. 

                    For Example:
                    
                        thisVeryLongSetOfWordsStrungTogether

The dollar sign $ and the underscore _ can also be used in names. They are regular symbols, just like letters, without any special meaning. 

                    For Example:
                    
                        let $ = 1;
                        let _ = 2;

                        console.log($+_);

Two variables that are spelt the same but have different letter case used are considered to be different variables. 

                    For Example:
                    
                        fire;
                        FIRE;

Non-Latin characters such as Cyrillic letters, Chinese logograms, and etc. are technically allowed, but there is an international convetion to use English in variable names. This also helps to preserve the work through conventional means and make it easier to use. 

                    For Example:
                    
                        let имя = '...';
                        let 我 = '...';

There are also sets of reserved words that can not be used. These are keywords that are used by the language itself. Some examples being let, class, return, and function. Attempting to use these words as variables will result in errors. 

                    For Example:
                    
                        let let = 5;
                        let return = 10;
