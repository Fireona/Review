How to run JavaScript code

All JavaScript written in the majority of the course will be run via browser. Later lessons in the Foundations and NodeJS path will show how to run JavaScript outside of the browser environment. This means that we will let the browser execute the code, even if it comes from a file we created.

Create a new HTML file and add the following script: 

                        
                            <script>
                                console.log("Hello, World!")
                            </script>
When you inspect the page and enter the Console tab of the DevTools, you will see the script pop up.

console.log is a command that directly prints something into the developer console of the browser.

Another method to include JavaScript into a webpage is with an external script. This is similar to linking an external CSS document to your website. 

                    
                        <script src="javascript.js"></script>

Just like how CSS documents have the extensions .css for stylesheets, JavaScript has the extension .js. External JavaScript files are used for more complex scripts.

External JavaScript files can be named freely into almost anything. So long as they have the .js extension, it will function. JavaScript files can be named anything, and can even be nameless (just .js).

Variables

Variables are the building blocks of any program. Think of them as "storage containers" for data in your code.

Variables can be declared using the let keyword. 

                    <script>
                    
                        let name = "John";
                        let surname = "Doe";

                        console.log(name);
                        console.log(surname);
                    
                    </script>

Inputting the above script will enter two logs into the console, one for the name and another for surname.

Variables can also be re-assigned. 

                    <script>
                    
                        let age = 11;
                        console.log(age);

                        age = 54;
                        
                        console.log(age);
                    
                    </script>

In the above script, the second declaration of age has omitted the variable let. This is because the variable had already been declared earlier, and you do not need it again when re-assigning values to variables.

When we have a constant that is never changed (like pi), we can use the keywordconst instead. 

                    <script>
                    
                        const pi = 3.14;
                        pi = 10;

                        console.log(pi);
                    
                    </script>

When inputting the above script, an error occurs. It does not reach the console log. This method of creating errors helps identify the location of said error and what is wrong.

In short, there are two methods of declaring a variable:

  • let, which can be re-assigned.
  • const, which can not be re-assigned.

In the past, the original method of declaring variables was the keyword var. However, there were issues when used that were fixed when let and const were introduced. Today, var is no longer used but you may still encounter it in other areas.

JavaScript.info on Variable Naming

Variable Naming

There are two limitations on variable names in JavaScript:

  • The name must contain only letters, digits, or the symbols $ or _.
  • The first character must not be a digit. 

                    For Example:
                    
                        let userName;
                        let test123;

Words go one after another, with each word starting with a capital letter with the exception of the first word. 

                    For Example:
                    
                        thisVeryLongSetOfWordsStrungTogether

The dollar sign $ and the underscore _ can also be used in names. They are regular symbols, just like letters, without any special meaning. 

                    For Example:
                    
                        let $ = 1;
                        let _ = 2;

                        console.log($+_);

Two variables that are spelt the same but have different letter case used are considered to be different variables. 

                    For Example:
                    
                        fire;
                        FIRE;

Non-Latin characters such as Cyrillic letters, Chinese logograms, and etc. are technically allowed, but there is an international convention to use English in variable names. This also helps to preserve the work through conventional means and make it easier to use. 

                    For Example:
                    
                        let имя = '...';
                        let 我 = '...';

There are also sets of reserved words that can not be used. These are keywords that are used by the language itself. Some examples being let, class, return, and function. Attempting to use these words as variables will result in errors. 

                    For Example:
                    
                        let let = 5;
                        let return = 10;

Constants

To declare a constant, use const. 

                    <script>
                    
                        const pi = 3.14
                        const g = 22/7
                    
                    <script/>

Variables declared using const are called constants and can not have values re-assigned. Any attempt doing so will result in an error.

A practice between developers is to assign constants to values that are difficult to remember. These constants are named using capital letters and underscores. Some examples are colors in hexadecimal values. 

                    <script>
                    
                        const COLOR_RED = "#F00";
                        const COLOR_GREEN = "#0F0";
                        const COLOR_BLUE = "00F";
                        const COLOR_ORANGE = "#FF7F00";

                        // ...when we need to pick a color
                        let color = COLOR_ORANGE;
                        alert(color);
                    
                    <script/>

In this scenario, using the const COLOR_ORANGE is much easier to handle than it's hexadecimal value "#FF7F00". It is easier to remember, type, and recognize.

There are rules to follow when using capitals in a constant. Some values are assigned before execution (before a page is loaded). There are also values are calculated first during the execution, but do not change after the initial assignment. 

                    <script>
                    
                        const pageLoadTime = 
                    
                    <script/>

The value of pageLoadTime is not known before a page is loaded, so it is named normally. However, because it is still a constant as it's value does not change once assigned. This means that capital-named constants are used for aliases that are "hard-coded" values.

Name things right

Any variable should have an alias that should be easily recognizable that describes the data it represents. A good example would be how easy a variable is recognized after doing something else for a period of time.

A general guideline:

  • Use human-readable names like userName or shoppingCart.
  • Avoid using abbreviations or short names such as x, b, c, etc. unless you know what you are doing.
  • Make names maximally descriptive and concise. Some bad examples are data and value because they don't tell use anything. It is only acceptable if the context makes it obvious what data or value they are referring to.
  • Terms should be agreed upon in mind and with the team. If a site visitor is called user then we should name related variables as currentUser or newUser rather than something oblivious as currentVisitor or newManInTown.

Summary

    Variable Naming
  • The name can only contain letters, digits, or the symbols $ or _.
  • The first character can not be a digit.
  • The first letter of all words in the target variable must be capitalized with the exception of the first.
  • The symbols $ and _ are symbols that have no meaning to them and can be used in names.
  • Two variables with the same spelling but with different letter cases are considered to be separate entities.
  • Non-Latin characters such as Cyrillic letters or Chinese logograms can be used as variables but should be frowned upon as it is conventional to use English and easier to understand.
  • Certain words can not be used as they are used as keywords in JavaScript. Some examples being:
    • let
    • class
    • return
    • function
    Constants
  • To assign a constant, use the keyword const.
  • Once assigned a value, it can not be changed.
  • Common practice is to use constants when dealing with values that are difficult to remember. Keep it simple, and assign names that make it easily to remember and recognize.
  • When naming constants, web developers follow a particular guideline.
      Use Uppercase (or ALL_CAPS) for constants that have:
    • Values known before execution.
    • Values that are hard-coded, never change, and used in multiple places.
    • Values that serve as configuration,settings or fixed references.
      Use camelCase (like regular variables) for constants that have:
    • Values that are computed or retrieved during execution.
    • Not meant to be reassigned but still not universal constants.
    Name things right
  • Any variable should have a name that accurately desribes the value it represents concisively.
    • Guideline:
    • Use human-readable names.
    • Don't use abbreviations or symbols such as x unless it is easy to identify the data it represents.
    • Don't use ambiguous terms such as data or value unless it is easy to identify the value and its context.
    • Make sure the term is agreeable in your head and with the team. Rather than creating a unique term for every visitor to a webpage such as thisVistor or newGuy, use something simple like newUser or currentUser.

Arithmetic Operations

  • Operations are defined by operators (variables and quantitative values) and operands:
    • *
    • /
    • +
    • -
  • Addition (+) 
                                    <script>
                                
                                    let x = 5;
                                    let y = 2;
                                    let z = x + y;
                                
                                <script/>
  • Subtraction (-) 
                                    <script>
                                
                                    let x = 5;
                                    let y = 2;
                                    let z = x - y;
                                
                                <script/>
  • Multiplication (*) 
                                    <script>
                                
                                    let x = 5;
                                    let y = 2;
                                    let z = x * y;
                                
                                <script/>
  • Division (/) 
                                    <script>
                                
                                    let x = 5;
                                    let y = 2;
                                    let z = x / y;                     
                                
                                <script/>
  • Remainder, also known as modulo operation (%) 
                                    <script>
                                
                                    let x = 5;
                                    let y = 2;
                                    let z = x % y;
                                
                                <script/>
  • Incrementation (++) 
                                    <script>
                                
                                    let x = 5;
                                    x++;
                                    let z = x;              
                                
                                <script/>
  • Decrementation (--) 
                                    <script>
                                
                                    let x = 5;
                                    x--;
                                    let z = x;
                                
                                <script/>
  • Exponentiation (**) 
                                    <script>
                                
                                    let x = 5;
                                    x**;
                                    let z = x;
                                
                                <script/>
